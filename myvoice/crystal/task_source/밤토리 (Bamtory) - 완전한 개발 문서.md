## 주요 기능

밤토리는 다음 핵심 기능을 제공합니다.

**일일 로그인 보너스**는 매일 첫 로그인 시 별과 하트를 보상으로 제공하며, 연속 로그인 일수에 따라 보상이 증가합니다.

**온보딩 튜토리얼**은 신규 사용자를 위한 5단계 가이드로, 앱 사용법을 쉽게 이해할 수 있도록 돕습니다.

**미션 시스템**은 다양한 학습 미션을 제공하며, 완료 시 XP와 별을 획득할 수 있습니다.

**어휘 학습**은 카테고리별로 단어를 학습하고, 발음 연습을 통해 실력을 향상시킬 수 있습니다.

**주간 목표**는 XP, 완료 미션, 학습 시간, 학습 단어 등 4가지 지표를 추적하여 학습 동기를 부여합니다.

**프로필 및 통계**는 사용자의 학습 기록과 통계를 시각화하여 성장을 확인할 수 있습니다.

**상점 시스템**은 획득한 별로 아이템을 구매하여 학습 경험을 개인화할 수 있습니다.

## 기술 스택

밤토리는 최신 웹 기술을 사용하여 구축됩니다.

### 프론트엔드

- **React 19**: UI 라이브러리
- **TypeScript 5.9**: 타입 안전성
- **Tailwind CSS 4**: 유틸리티 우선 스타일링
- **shadcn/ui**: 재사용 가능한 컴포넌트
- **Wouter**: 경량 라우팅
- **tRPC 11**: 타입 안전 API 호출

### 백엔드

- **Node.js 22**: 런타임
- **Express 4**: 웹 프레임워크
- **tRPC 11**: API 레이어
- **Drizzle ORM**: 데이터베이스 ORM
- **MySQL/TiDB**: 데이터베이스

### 개발 도구

- **Vite**: 빌드 도구
- **Vitest**: 테스트 프레임워크
- **ESLint**: 코드 린팅
- **Prettier**: 코드 포맷팅

## 개발 워크플로우

밤토리 개발 시 다음 워크플로우를 따릅니다.

### 1. 기능 개발

새로운 기능을 개발할 때는 다음 순서로 진행합니다.

1. **데이터베이스 스키마 정의** (`drizzle/schema.ts`)
2. **마이그레이션 실행** (`pnpm db:push`)
3. **데이터베이스 헬퍼 함수 작성** (`server/db.ts`)
4. **API 엔드포인트 구현** (`server/routers.ts`)
5. **프론트엔드 UI 구현** (`client/src/pages/`)
6. **테스트 작성** (`server/*.test.ts`)
7. **테스트 실행** (`pnpm test`)

### 2. 코드 리뷰

Pull Request를 생성하기 전에 다음을 확인합니다.

- [ ] TypeScript 오류 없음
- [ ] ESLint 경고 없음
- [ ] Prettier로 포맷팅 완료
- [ ] 모든 테스트 통과
- [ ] 문서 업데이트 완료

### 3. 배포

배포 전에 "배포 체크리스트"를 모두 확인합니다.

## 자주 묻는 질문

### Q: 왜 tRPC를 사용하나요?

A: tRPC는 타입 안전성을 제공하여 프론트엔드와 백엔드 간의 타입 불일치를 컴파일 타임에 잡을 수 있습니다. REST API와 달리 별도의 API 문서나 클라이언트 코드 생성이 필요 없습니다.

### Q: 왜 Drizzle ORM을 사용하나요?

A: Drizzle은 TypeScript 우선 ORM으로, 타입 안전성과 성능을 모두 제공합니다. Prisma보다 가볍고 빠르며, SQL과 유사한 문법을 사용하여 학습 곡선이 낮습니다.

### Q: 모바일 앱도 만들 수 있나요?

A: 현재 문서는 웹 앱을 대상으로 하지만, React Native로 모바일 앱을 만들 수 있습니다. tRPC와 대부분의 비즈니스 로직을 재사용할 수 있습니다.

### Q: 다국어 지원은 어떻게 하나요?

A: `react-i18next` 라이브러리를 사용하여 다국어를 지원할 수 있습니다. 모든 텍스트를 번역 키로 대체하고, 언어별 JSON 파일을 작성합니다.

### Q: 오프라인 모드는 어떻게 구현하나요?

A: Service Worker와 IndexedDB를 사용하여 오프라인 모드를 구현할 수 있습니다. Workbox 라이브러리를 사용하면 쉽게 구현할 수 있습니다.

## 기여 가이드

밤토리 프로젝트에 기여하려면 다음 가이드라인을 따라주세요.

### 코드 스타일

- TypeScript를 사용하고, `any` 타입은 피합니다.
- Tailwind CSS 유틸리티 클래스를 우선 사용합니다.
- 컴포넌트는 작고 재사용 가능하게 작성합니다.
- 함수는 단일 책임 원칙을 따릅니다.

### 커밋 메시지

커밋 메시지는 다음 형식을 따릅니다.

```
<type>: <subject>

<body>
```

**타입:**
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 스타일 변경
- `refactor`: 리팩토링
- `test`: 테스트 추가/수정
- `chore`: 빌드 설정 변경

**예시:**
```
feat: 일일 로그인 보너스 시스템 구현

- 연속 로그인 일수 추적
- 보상 계산 로직 구현
- 보너스 모달 UI 추가
```

### Pull Request

PR을 생성할 때는 다음을 포함합니다.

- 변경 사항 요약
- 스크린샷 (UI 변경 시)
- 테스트 결과
- 관련 이슈 번호
