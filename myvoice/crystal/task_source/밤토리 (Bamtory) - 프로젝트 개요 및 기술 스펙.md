## 프로젝트 개요

밤토리는 Duolingo 스타일의 모바일 최적화 영어 학습 애플리케이션입니다. 사용자는 Luna라는 캐릭터를 도우며 미션을 완료하고, 어휘를 학습하며, 일일 목표를 달성하는 게임화된 학습 경험을 제공받습니다.

### 핵심 가치 제안

밤토리는 학습을 게임처럼 재미있게 만들어 사용자의 지속적인 참여를 유도합니다. 연속 로그인 스트릭, 별과 하트 보상 시스템, 주간 목표 추적을 통해 사용자는 학습 습관을 형성하고 성취감을 느낄 수 있습니다.

### 주요 기능

밤토리는 다음과 같은 핵심 기능을 제공합니다.

**인증 및 온보딩 시스템**은 Manus OAuth 기반의 안전한 로그인을 제공하며, 신규 사용자를 위한 5단계 인터랙티브 튜토리얼을 통해 서비스 사용법을 안내합니다. 비로그인 사용자는 자동으로 로그인 페이지로 리다이렉트되어 서비스 보안을 강화합니다.

**일일 로그인 보너스 시스템**은 매일 첫 로그인 시 별과 하트를 보상으로 제공합니다. 연속 로그인 일수에 따라 보상이 증가하며(1일차: 5별/1하트 → 7일차 이상: 20별/3하트), 하루에 한 번만 청구할 수 있습니다. 아름다운 그라디언트 모달과 슬라이드 애니메이션으로 보상을 표시합니다.

**미션 시스템**은 사용자가 Luna를 도와 다양한 영어 학습 미션을 완료할 수 있도록 합니다. 각 미션은 난이도, 소요 시간, 보상(별)이 명시되어 있으며, 완료 시 XP와 별을 획득합니다. 미션 유형에는 "Luna Meets Friends", "Luna is Hungry", "Luna is Thirsty" 등이 있습니다.

**어휘 학습 시스템**은 카테고리별로 구성된 어휘 학습 기능을 제공합니다. 사용자는 학습한 단어 수를 추적하고, 주간 목표에 반영할 수 있습니다.

**주간 목표 추적**은 사용자의 학습 활동을 4가지 지표로 추적합니다. 획득 XP, 완료 미션 수, 학습 시간, 학습 단어 수를 시각화하여 진행 상황을 한눈에 파악할 수 있습니다.

**프로필 및 통계**는 사용자의 레벨, XP, 연속 로그인 일수, 보유 별과 하트를 표시합니다. 주간 학습량을 막대 그래프로 시각화하고, 학습 통계(완료 미션, 학습 시간, 어휘 수, 발음 정확도)를 제공합니다. 배지 시스템을 통해 사용자의 업적을 표시합니다.

**상점 시스템**은 사용자가 획득한 별을 사용하여 아이템을 구매할 수 있는 기능을 제공합니다. 하트 충전, 스트릭 복구, 아바타 변경 등의 아이템이 있습니다.

## 아키텍처 개요

밤토리는 클라이언트-서버 아키텍처를 기반으로 하며, tRPC를 통해 타입 안전한 통신을 구현합니다.

### 디렉토리 구조

```
bamtory/
├── client/                 # 프론트엔드 애플리케이션
│   ├── public/            # 정적 자산
│   └── src/
│       ├── _core/         # 핵심 유틸리티 (useAuth 등)
│       ├── components/    # 재사용 가능한 컴포넌트
│       │   ├── ui/       # shadcn/ui 컴포넌트
│       │   ├── layout/   # 레이아웃 컴포넌트
│       │   ├── mission/  # 미션 관련 컴포넌트
│       │   └── ...
│       ├── contexts/      # React 컨텍스트
│       ├── hooks/         # 커스텀 훅
│       ├── lib/           # 라이브러리 설정
│       ├── pages/         # 페이지 컴포넌트
│       ├── App.tsx        # 라우팅 설정
│       └── main.tsx       # 엔트리 포인트
├── server/                # 백엔드 애플리케이션
│   ├── _core/            # 핵심 서버 로직
│   │   ├── context.ts    # tRPC 컨텍스트
│   │   ├── trpc.ts       # tRPC 설정
│   │   ├── oauth.ts      # OAuth 처리
│   │   └── ...
│   ├── trpc/
│   │   └── routers/      # tRPC 라우터
│   ├── db.ts             # 데이터베이스 헬퍼
│   ├── routers.ts        # 메인 라우터
│   └── *.test.ts         # 테스트 파일
├── drizzle/              # 데이터베이스 스키마 및 마이그레이션
│   ├── schema.ts         # 데이터베이스 스키마
│   └── migrations/       # 마이그레이션 파일
├── shared/               # 공유 타입 및 상수
└── package.json          # 의존성 관리
```

### 데이터 흐름

사용자 요청은 다음과 같은 흐름으로 처리됩니다.

1. **클라이언트**: 사용자가 UI에서 액션을 수행합니다 (예: 로그인 보너스 청구)
2. **tRPC 클라이언트**: `trpc.dailyBonus.claim.useMutation()`을 호출합니다
3. **tRPC 서버**: 요청을 받아 인증을 확인하고 비즈니스 로직을 실행합니다
4. **데이터베이스**: Drizzle ORM을 통해 MySQL 데이터베이스에 접근합니다
5. **응답**: 타입 안전한 응답을 클라이언트로 반환합니다
6. **UI 업데이트**: React Query가 캐시를 업데이트하고 UI를 자동으로 갱신합니다

### 인증 흐름

밤토리는 Manus OAuth를 사용하여 안전한 인증을 구현합니다.

1. **로그인 요청**: 사용자가 로그인 버튼을 클릭하면 `getLoginUrl()`을 통해 OAuth 서버로 리다이렉트됩니다
2. **OAuth 인증**: 사용자가 Manus OAuth 포털에서 인증을 완료합니다
3. **콜백 처리**: `/api/oauth/callback` 엔드포인트가 인증 코드를 받아 세션 쿠키를 생성합니다
4. **세션 유지**: 모든 API 요청에 세션 쿠키가 포함되어 사용자 인증을 확인합니다
5. **컨텍스트 주입**: `server/_core/context.ts`가 각 요청에 `ctx.user`를 주입합니다
6. **보호된 프로시저**: `protectedProcedure`가 인증되지 않은 요청을 차단합니다

## 개발 환경 설정

밤토리 개발 환경을 설정하려면 다음 단계를 따르세요.

### 사전 요구사항

- Node.js 22.13.0 이상
- pnpm 패키지 매니저
- MySQL 데이터베이스 (또는 TiDB)

### 설치 및 실행

```bash
# 의존성 설치
pnpm install

# 데이터베이스 마이그레이션
pnpm db:push

# 개발 서버 실행
pnpm dev

# 테스트 실행
pnpm test

# 프로덕션 빌드
pnpm build
```

### 환경 변수

밤토리는 다음과 같은 환경 변수를 사용합니다. 이 변수들은 Manus 플랫폼에서 자동으로 주입됩니다.

| 변수명 | 설명 |
|--------|------|
| `DATABASE_URL` | MySQL/TiDB 연결 문자열 |
| `JWT_SECRET` | 세션 쿠키 서명 비밀키 |
| `VITE_APP_ID` | Manus OAuth 애플리케이션 ID |
| `OAUTH_SERVER_URL` | Manus OAuth 백엔드 URL |
| `VITE_OAUTH_PORTAL_URL` | Manus 로그인 포털 URL |
| `OWNER_OPEN_ID` | 소유자 Open ID |
| `OWNER_NAME` | 소유자 이름 |

## 코드 규칙 및 베스트 프랙티스

밤토리 프로젝트는 다음과 같은 코드 규칙을 따릅니다.

### TypeScript 규칙

모든 코드는 TypeScript로 작성되며, `any` 타입 사용을 최소화합니다. tRPC를 통해 클라이언트와 서버 간 타입 안정성을 보장합니다.

### React 규칙

함수형 컴포넌트와 Hooks를 사용하며, `useState`, `useEffect`, `useMemo` 등을 적절히 활용합니다. 렌더 단계에서 `setState`나 네비게이션을 호출하지 않으며, 필요시 `useEffect`로 감싸줍니다.

### 스타일링 규칙

Tailwind CSS 유틸리티 클래스를 우선 사용하며, 커스텀 CSS는 `client/src/index.css`에 정의합니다. shadcn/ui 컴포넌트를 활용하여 일관된 디자인을 유지합니다.

### 파일 명명 규칙

- 컴포넌트: PascalCase (예: `DailyBonusModal.tsx`)
- 유틸리티: camelCase (예: `getLoginUrl.ts`)
- 페이지: PascalCase (예: `Home.tsx`)
- 테스트: `*.test.ts` 형식

### 커밋 메시지 규칙

커밋 메시지는 다음 형식을 따릅니다.

```
<type>: <subject>

<body>
```

타입은 `feat` (새 기능), `fix` (버그 수정), `docs` (문서), `style` (포맷팅), `refactor` (리팩토링), `test` (테스트), `chore` (기타) 중 하나를 사용합니다.

## 배포 및 운영

밤토리는 Manus 플랫폼의 내장 호스팅을 사용하여 배포됩니다.

### 배포 프로세스

1. **체크포인트 생성**: `webdev_save_checkpoint`를 통해 현재 상태를 저장합니다
2. **Management UI**: 사용자가 Publish 버튼을 클릭하여 배포를 시작합니다
3. **자동 배포**: Manus 플랫폼이 자동으로 빌드하고 배포합니다
4. **커스텀 도메인**: Settings > Domains에서 커스텀 도메인을 설정할 수 있습니다

### 모니터링

Management UI의 Dashboard 패널에서 다음 지표를 모니터링할 수 있습니다.

- UV/PV (방문자 및 페이지뷰)
- 서버 상태 및 로그
- 데이터베이스 연결 상태

## 문제 해결

개발 중 자주 발생하는 문제와 해결 방법을 정리했습니다.

### 개발 서버가 응답하지 않음

`webdev_restart_server`를 통해 서버를 재시작합니다. 브라우저 캐시를 삭제하거나 시크릿 모드로 접속해봅니다.

### TypeScript 오류

`pnpm install`로 의존성을 재설치하고, VSCode를 재시작합니다. `node_modules`와 `.vite` 디렉토리를 삭제 후 재설치합니다.

### 데이터베이스 연결 오류

`DATABASE_URL` 환경 변수가 올바르게 설정되었는지 확인합니다. `pnpm db:push`로 스키마를 동기화합니다.

### 인증 오류

세션 쿠키가 올바르게 설정되었는지 확인합니다. 브라우저가 쿠키를 차단하지 않는지 확인합니다 (Safari Private Browsing, Firefox Strict ETP 등은 지원하지 않음).
