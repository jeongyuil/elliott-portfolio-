VIP Scope
1. **서비스 명칭: 마을 (Maeul)**
	1. 작은 단위의 가구 공동체 (면,읍,리)
	2. 커뮤니티, 부동산 등을 지키기 위한 공동체
	3. 부동산 정보로 시작해서 향후에 커뮤니티에서 필요로 하는 정보로 확장 가능
	4. 정보 공유 커뮤니티로 트래픽을 모으고, 향후 지역 광고 플랫폼(오프라인)으로 확장
2. 이용자 정의
	1. 서울 아파트 커뮤니티 (단톡방)에 들어가 있는 아파트 거주자 또는 구매 예정자
	2. 30대 - 40대 아이가 있는 가정 구성원 (부모)
		1. 영아기 부모
		2. 유아기 부모
		3. 초등학교 저학년 부모
3. 1차 VIP 개발 스펙
	1. **이용자에게 전달할 1차 콘텐츠** - 1차 개발 VIP
	2. **콘텐츠를 통해 유입될 랜딩 페이지 2차 콘텐츠**  - 우리 서비스 가입 유도 페이지
4. 1차 서비스 범위: App & Web
5. 1차 개발 범위
	1. 데이터 수집
		1. 외부 데이터, API 사용해서 수집: 가능 리스트 필요
		2. 매물 정보는 저작권 이슈 발생 가능: 법적인 검토 필요 (상업적인 목적으로 이용 불가능? 우회 방법은?)
	2. User Journey Map Design
		1. 이용자 경험 순서에 따라 들어가야 할 기능들을 먼저 정의해보기
			1. **카카오톡 (단톡방) -> URL Landing Page** : 확정
			2. 어떤 화면(?)/ 어떤 정보(?) --> 가입 유도 (?) : 나중에 정하기
		2. 이용자에게 얻을만한 정보
			1. 예측 값: 내가 중요하게 생각하는 것 (교육/ 병원/ 소비/ 무엇?) --> 제휴업체로 비즈니스 모델로 가져가는게 어떨까?
			2. 확정 값: 위치 or 아파트 이름 등
		3. 내가 살고 있는 아파트 정보 vs. 내가 가고 싶은 아파트 정보
6. 2차 개발 스펙: 우리 서비스
	1. 데이터 모델링 - 정형, 비정형, ... (TBD)
	2. AI 에이전트 학습 (TBD)
	3. 서비스: VIP 서비스 개발하면서 아이디어가 나올 것 같다.
		1. 이용자들에게 어떤 핵심 서비스가 필요하지?
		2. 인간 대신 가치있는 작업을 해야하는 부분이 무엇이지?
7. 2차 개발 서비스 관련 아이디어
	1. 단톡방마다 나오는 주제에 대해 분석해서 사람들이 관심 있어 할만한 정보를 제공 하는건 어떨까? (Personalization) - 사회, 경제/ 육아 정보/ 부동산 매물
	2. Feasibility
		1. 카톡 단톡방에서 데이터 받아올 수 있는지 구현 가능성 확인
