# VOICE AI 교육 플랫폼 MVP 개발 로드맵

본 문서는 `VOICE AI 교육 플랫폼 사업 기획서`를 바탕으로, 1분기(Q1) 내 MVP(Minimum Viable Product) 출시를 위한 구체적인 개발 로드맵을 정의합니다.

## 1. MVP 목표 및 범위

### 1.1 목표
- **핵심 가치 검증:** AI와의 자연스러운 음성 대화가 아동의 흥미를 유발하고 지속 가능한지 검증
- **데이터 파이프라인 구축:** 음성 데이터 수집, 텍스트 변환, 저장, 분석의 기술적 타당성 확보
- **시장 진입:** 초기 베타 테스터(100가정) 모집을 위한 서비스 안정성 확보

### 1.2 MVP 포함 기능 (Scope)
| 구분 | 기능 | 상세 내용 | 비고 |
| :--- | :--- | :--- | :--- |
| **사용자(아동)** | AI 음성 대화 | 실시간 음성 대화 (STT-LLM-TTS) | Latency 최적화 필수 |
| | 캐릭터 선택 | 대화 상대(페르소나) 선택 기능 | 기본 2~3종 제공 |
| **사용자(부모)** | 대시보드 | 자녀 학습 현황 요약 (발화량, 시간) | |
| | 리포트 (Basic) | 월간 학습 리포트 (PDF 다운로드) | 텍스트 위주 |
| **관리자** | 회원 관리 | 테스터 계정 관리 | |
| | 데이터 모니터링 | 대화 로그 및 오류 모니터링 | |

## 2. 개발 일정 (총 12주 / 3개월)

### Phase 1: 기획 및 프로토타이핑 (Week 1-2)
- **기획:**
    - 핵심 페르소나(AI 캐릭터) 및 대화 시나리오 정의 (연령별 1~2개)
    - 앱 UI/UX 와이어프레임 및 디자인 (아동용/부모용 분리)
    - 데이터베이스 스키마 설계 (User, Session, Message, Report)
- **R&D:**
    - OpenAI Realtime API 및 Whisper 모델 성능 테스트
    - 음성 입출력 Latency 최소화 기술 검증

### Phase 2: 핵심 엔진 및 백엔드 개발 (Week 3-6)
- **AI 엔진 개발:**
    - LangChain 기반 대화 관리자(Dialogue Manager) 구현
    - STT(Whisper) -> LLM -> TTS 파이프라인 구축
    - 프롬프트 엔지니어링 (아동 눈높이 대화, 비속어 필터링)
- **백엔드 개발:**
    - REST API 서버 구축 (Python/FastAPI 권장)
    - 사용자 인증/인가 (Auth) 시스템
    - 대화 로그 적재 및 비동기 처리 시스템 (Celery/Redis)

### Phase 3: 프론트엔드 개발 (Week 5-9)
- **앱 개발 (Flutter/React Native):**
    - 회원가입/로그인 및 프로필 설정
    - **[핵심]** 실시간 음성 대화 인터페이스 (애니메이션, 마이크 제어)
    - 부모용 대시보드 및 리포트 뷰어 구현
- **연동:**
    - 앱-서버 간 API 연동 및 소켓 통신(WebSocket) 구현

### Phase 4: 분석 시스템 및 리포트 (Week 8-10)
- **데이터 분석:**
    - 대화 로그 기반 기초 통계 산출 (발화 시간, 턴 수, 어휘 수)
    - 리포트 생성 로직 구현 (데이터 집계 -> 텍스트 요약 -> PDF 변환)
- **관리자 도구:**
    - 간단한 어드민 페이지 구축

### Phase 5: 테스트 및 안정화 (Week 11-12)
- **QA (Quality Assurance):**
    - 기능 테스트 및 버그 수정
    - 음성 인식률 및 응답 속도 최적화
    - 보안 점검 (개인정보 암호화 확인)
- **배포:**
    - 클라우드 인프라 구축 (AWS/GCP)
    - 앱 스토어/플레이 스토어 심사 제출

## 3. 기술 스택 (제안)

- **Frontend:** Flutter (iOS/Android 동시 대응)
- **Backend:** Python (FastAPI)
- **AI/ML:** 
    - LLM: GPT-4o (or GPT-4o-mini for cost efficiency)
    - STT: Whisper (OpenAI API)
    - TTS: ElevenLabs or OpenAI TTS
    - Framework: LangChain
- **Database:** PostgreSQL (관계형 데이터), Redis (캐싱/메시지큐), S3 (음성 파일/PDF 저장)
- **Infra:** AWS (EC2, RDS, ElastiCache)

## 4. 필요 인력 구성 (MVP 기준)
*사업계획서 9.2 조직 구성 참조*

1. **PM/기획 (CEO 겸임):** 요구사항 정의, 일정 관리
2. **CTO/Backend:** 아키텍처 설계, 서버 및 AI 파이프라인 개발
3. **Frontend Dev:** 모바일 앱 개발
4. **AI/Data Dev:** 프롬프트 엔지니어링, 데이터 분석 로직 구현

## 5. 주요 체크포인트 (Milestones)

- **M1 (Week 4):** 사내 데모 (음성으로 AI와 핑퐁 대화 가능)
- **M2 (Week 8):** 알파 버전 (앱 기본 기능 완료, 내부 테스트 시작)
- **M3 (Week 12):** 베타 출시 (스토어 등록, 베타 테스터 100명 모집 시작)
