## 1. 목적 및 범위

### 1.1 목적

본 문서는 VOICE AI 교육 플랫폼에서 사용하는 핵심 개념(엔티티), 관계, 속성 및 비즈니스 룰을 온톨로지(도메인 모델) 형태로 정의한다. 이를 통해:

- 서비스/데이터/AI 팀 간 동일한 도메인 이해 기반 확보  
      
    
- 그래프/온톨로지 기반 데이터베이스(Knowledge Graph) 설계 기준 제공  
      
    
- AI 분석/추천/진단 모델 설계 시 공통 언어 제공  
      
    
- 교육/언어 발달 전문가의 검증 대상 표준 모델 제공  
      
    
- 추후 데이터 거버넌스/프라이버시/모델 버전 관리를 위한 확장 포인트 확보  
      
    

### 1.2 적용 범위

#### 서비스 범위

- 4~12세 아동 대상 AI 음성 기반 1:1 대화 세션  
      
    
- 모바일 앱 및 IoT 디바이스를 통한 B2C 구독형 서비스  
      
    
- 월간/연간 구독 요금제와 정기 피드백 리포트 제공  
      
    

#### 데이터 범위

- 이용자 프로필 – 아동 / 보호자 (별도 구분)  
      
    
- 음성 대화 세션 및 발화 (Speech To Text - STT 결과 포함)  
      
    

- 아이와 AI가 대화한 한 번의 수업/놀이 시간(세션) 전체와,  
    그 안에서 오고간 모든 말 한 마디 한 마디(발화)를,  
    음성 + 음성을 인식해 텍스트로 변환한 결과까지 같이 저장한다.  
      
    
- 세션(session)  
      
    

- 정의: 아이와 AI가 한 번에 쭉 대화한 한 덩어리  
      
    
- 세션에 들어가는 것들:  
      
    

- 시작 시간 / 끝나는 시간  
      
    
- 어떤 아이가 했는지  
      
    
- 어떤 커리큘럼(동화, 게임 등)을 했는지  
      
    
- 세션 전체 참여도/집중도 같은 요약 점수  
      
    

- 발화(utterance)  
      
    

- 정의: 세션 안에서 한 번 말한 것을 1개의 단위  
      
    

- 음성 데이터 저장 전략  
      
    

- 언어 교육/평가 측면에서 음성 데이터의 원본 가치가 매우 높음  
    (발음 문제, 말 더듬, 구어적 특성, 억양, 감정 등)  
      
    
- 샘플링 전략(예시):  
      
    

- 첫 달 첫 세션  
      
    
- 매 달 1~2개 대표 세션  
      
    
- 전문가/시스템이 “주의 필요”로 플래그한 세션  
      
    
- 위 세션의 full audio는 장기 보관  
      
    

- 관련 데이터: Session 엔티티의 필드  
      
    

- audio_retention_level (enum: none, partial, full)  
      
    
- flag_for_expert_review (bool)  
      
    

- 구독/요금제/이용 패턴 관련 메타데이터 (예시)  
      
    

- subscription_model: retail / wholesale  
      
    
- subscription_plan: basic / standard / premium  
      
    
- renewal_period: monthly, annual  
      
    
- subscription_start_date: yyyy-mm-dd  
      
    
- renewal_date: yyyy-mm-dd  
      
    
- subscription_status: paid_good_standing, paid_pending_cancel, grace, hold, stopped, trial  
      
    
- streak_tenure: 1,2,3, …  
      
    

- 학습/놀이 커리큘럼 및 활동  
      
    
- 학습/놀이 성취도 지표  
      
    

- zzz분 대화 달성 (engagement - 사용시간)  
      
    
- xyz 수준 달성 (engagement - 뱃지)  
      
    
- (xx 단계 중) yy% 달성 (engagement - 달성률 or Rank)  
      
    

- 언어/인지 발달 지표 및 Skill  
      
    

- 언어학적 구체화된 평가 기준에 따라 월간 리포트 내용과 Skill을 직접적으로 연결  
      
    

- 월간 리포트 및 평가 결과  
      
    

### 1.3 Enum 네이밍 규칙

- enum 값은 snake_case 영문 소문자를 사용한다.  
      
    
- 공백은 사용하지 않고 '_'로 대체한다.  
      
    
- 예: paid_good_standing, wh_question_answer, pdf_ai_video_expert  
      
    

### 1.4 데이터 거버넌스 / 프라이버시 원칙 (요약)

- Child / FamilyAccount / raw_audio_path / STT 텍스트 등은 PII/민감 정보로 분류된다.  
      
    
- 각 엔티티/필드 수준에서 민감도 태그(PII/Sensitive) 부여를 고려하며,  
    실제 구현 시:  
      
    

- 데이터 마스킹,  
      
    
- 접근 권한(역할 기반),  
      
    
- 보존 기간(retention 정책)  
    을 설정할 수 있도록 설계한다.  
      
    

- Consent(동의) 관련 엔티티(예: ConsentRecord)는 후속 버전에서 추가하며,  
      
    

- 어떤 보호자가  
      
    
- 어떤 목적(학습, 리포트, 연구 등)으로  
      
    
- 어떤 데이터 처리(음성 보존, 모델 학습 등)에  
    동의했는지 추적 가능하도록 확장한다.

## 2. 핵심 엔티티 정의

형식:

- 엔티티명 (한국어 / 영어 Class 이름)  
      
    
- 설명  
      
    
- 주요 속성 (필수/선택, 타입, 설명)  
      
    

---

### 2.1 Child (아동)

설명  
서비스의 실제 학습 사용자. 4~12세 아동. 언어 발달, 인지 발달 상태를 추적하는 주체.

주요 속성

- child_id (PK, string): 아동 고유 식별자  
      
    
- name (string): 이름 (가명 가능)  
      
    
- birth_date (date): 생년월일  
      
    
- age (int): 만 나이 (파생 가능)  
      
    
- gender (enum: male, female, other)  
      
    
- primary_language (enum: ko, en, etc.)  
      
    
- secondary_language (enum, nullable)  
      
    
- development_stage_language (enum): 언어 발달 단계 (예: 지연/평균/고도 등)  
      
    
- development_stage_cognitive (enum, optional)  
      
    
- preferences_topics (array): 선호 주제 (동물, 공룡, 우주 등)  
      
    
- created_at / updated_at (datetime)  
      
    

---

### 2.2 Parent / FamilyAccount (보호자 / 가족 계정)

설명  
결제 및 서비스 구독의 실 사용자. 1개 FamilyAccount 아래 여러 Child 연결 가능. 타겟은 4~12세 자녀를 둔 학부모.

주요 속성

- family_id (PK, string)  
      
    
- parent_name (string)  
      
    
- contact_email (string)  
      
    
- contact_phone (string)  
      
    
- address_region (string): 지역/도시  
      
    
- primary_child_id (FK, optional)  
      
    
- signup_date (date)  
      
    
- preferred_report_tier (enum: basic, standard, premium)  
      
    
- marketing_opt_in (bool)  
      
    
- status (enum: active, paused, churned)  
      
    

---

### 2.3 SubscriptionPlan (구독 플랜)

설명  
과금/이용 한도를 정의하는 요금제. Basic/Standard/Premium 플랜.

주요 속성

- plan_id (PK, string)  
      
    
- plan_name (enum: basic, standard, premium)  
      
    
- subscription_model (enum: retail, wholesale)  
      
    
- billing_platform (enum: direct, apple, google, sk, kt)  
      
    
- monthly_price (int, KRW)  
      
    
- allowed_sessions_per_week (int or “unlimited”)  
      
    
- report_type (enum: pdf_only, pdf_ai_video, pdf_ai_video_expert)  
      
    
- extra_features (JSON/array): 부가 서비스(전문가 상담, 맞춤 커리큘럼 등)  
      
    
- status (enum: active, deprecated)  
      
    

---

### 2.4 Subscription (구독 인스턴스)

설명  
특정 FamilyAccount가 특정 Plan을 사용하는 실제 구독 기록.

주요 속성

- subscription_id (PK, string)  
      
    
- family_id (FK → FamilyAccount)  
      
    
- plan_id (FK → SubscriptionPlan)  
      
    
- start_date / end_date (date)  
      
    
- billing_cycle (enum: monthly, annual)  
      
    
- status (enum: paid_good_standing, paid_pending_cancel, grace, hold, stopped, trial)  
      
    
- cancel_reason (string/enum, nullable)  
      
    
- auto_renew (bool)

### 2.5.1 Session (학습/놀이 대화 세션)

설명  
AI 튜터와 아동의 1:1 음성 대화 세션. 1회 20분 내외, 주 3~5회 권장.

주요 속성

- session_id (PK, string)  
      
    
- child_id (FK → Child)  
      
    
- subscription_id (FK → Subscription)  
      
    
- start_time / end_time (datetime)  
      
    
- duration_seconds (int)  
      
    
- session_type (enum: free_talk, curriculum, assessment, game)  
      
    
- language_mode (enum: ko_only, en_only, mixed)  
      
    
- device_type (enum: mobile, tablet, smart_speaker, etc.)  
      
    
- curriculum_unit_id (FK, nullable)  
      
    
- engagement_score (float, 0~1): 참여도  
      
    
- attention_score (float, 0~1): 집중도  
      
    
- emotional_state_overall (enum: positive, neutral, negative)  
      
    
- raw_audio_path (string): S3 등 저장 경로  
      
    
- stt_processed (bool)  
      
    
- audio_retention_level (enum: none, partial, full)  
      
    
- flag_for_expert_review (bool)  
      
    

---

### 2.5.2 SessionActivity (세션-활동 이력)

설명  
해당 세션에서 어떤 Activity들이 실제로 수행되었는지 확인하기 위함.  
아이 반응에 따라 중간에 스킵/반복/분기하여 Activity 순서가 바뀔 수 있고, A/B 테스트에서 Activity 조합을 변경할 수 있으므로 CurriculumUnit만으로는 실제 수행 이력 추적이 안 됨.

주요 속성

- session_activity_id (PK, string)  
      
    
- session_id (FK → Session)  
      
    
- activity_id (FK → Activity)  
      
    
- order_index (int): 세션 내 실행 순서  
      
    
- started_at (datetime, optional)  
      
    
- ended_at (datetime, optional)  
      
    
- status (enum: completed, skipped, aborted)  
      
    

---

### 2.6.1 Utterance (발화)

설명  
STT 처리된 한 번의 말하기 단위. 발화자(아동/AI), 텍스트, 발음/언어 지표 등을 포함.

주요 속성

- utterance_id (PK, string)  
      
    
- session_id (FK → Session)  
      
    
- session_activity_id (FK → SessionActivity, nullable): 이 발화는 세션 내 어떤 Activity 수행 중 나온 것인가  
      
    
- turn_index (int): 세션 내 발화 순서  
      
    
- speaker_type (enum: child, ai)  
      
    

텍스트 / STT 관련

- text_raw (string): STT 엔진이 반환한 원문 그대로  
      
    
- text_normalized (string, optional): 후처리(문장부호 복원, 정규화 등)된 텍스트  
      
    
- is_text_normalized (bool): normalized 필드가 적용되었는지 여부  
      
    
- language (enum: ko, en, etc.): 발화의 주요 언어  
      
    
- dominant_language (enum: ko, en, mixed): 코드 스위칭 상황 포함한 지배적 언어  
      
    
- language_mix_ratio_ko (float, 0~1, optional)  
      
    
- language_mix_ratio_en (float, 0~1, optional)  
      
    

STT / 스코어링 엔진 메타데이터

- stt_engine (enum: gcp, aws, ncp, custom 등)  
      
    
- stt_model_version (string)  
      
    
- stt_confidence (float, 0~1): 문장 단위 신뢰도  
      
    
- scoring_model_version (string): pronunciation_score, fluency_score 산출 모델 버전  
      
    

언어적 특징

- word_count (int)  
      
    
- unique_word_count (int): 어휘 다양성  
      
    
- avg_sentence_length (float): 문장 구조 복잡도  
      
    
- pronunciation_score (float, 0~1)  
      
    
- fluency_score (float, 0~1)  
      
    
- scoring_reference (enum: native_en, korean_learner_en 등): 어떤 기준으로 점수를 해석하는지  
      
    
- emotion_label (enum: happy, sad, angry, neutral, etc.)  
      
    
- topic_tags (array)  
      
    

스킬/오류 태깅

- skill_ids (array<FK → Skill>): 이 발화가 측정/훈련하는 Skill들  
      
    
- error_categories (array<FK → ErrorCategory>): 발화 내 오류 유형  
      
    

대화 상의 역할

- utterance_role (enum: prompt, instruction, feedback, filler 등):  
      
    

- AI 발화의 역할 – 아이의 응답을 유도, 과제/지시, 칭찬/교정/평가, 분위기 전환 등  
      
    

- expected_child_response_type (enum: yes_no, wh_answer, choice_selection, repetition 등):  
      
    

- 해당 AI 발화에 기대한 아동 응답의 유형  
      
    

[Note] 언어 교육 관점 고려사항

- utterance_role, expected_child_response_type는 주로 speaker_type = 'ai'인 발화에 사용되며,  
    아동 발화(speaker_type = 'child')의 경우 대부분 null 또는 not_applicable로 처리한다.  
      
    
- 이 정보는 “아이가 의도된 질문을 이해했는가?”를 판단하는데 활용된다.

### 2.6.2 TaskDefinition (과제 정의) – optional (향후 확장용)

설명  
하나의 Activity 안에 여러 개의 문항/질문 템플릿이 있을 수 있으며, TaskAttempt는 이 정의를 참조한다. v0.1.1에서는 Optional로 정의하되, 추후 확장 시 사용 가능.

주요 속성

- task_definition_id (PK, string)  
      
    
- activity_id (FK → Activity)  
      
    
- prompt_template (text): AI가 사용할 질문/과제 템플릿  
      
    
- target_skill_ids (array<FK → Skill>)  
      
    
- expected_response_pattern (enum: description, wh_question_answer, repetition, narrative, opinion, role_play 등)  
      
    
- difficulty_level (int/enum, optional)  
      
    

---

### 2.6.3 TaskAttempt (과제수행)

설명  
Utterance만으로 측정되기 어려운 “과제 수행 정확도(정답 여부)”와 “언어적 복잡성/발음”을 분리해서 볼 수 있게 하는 엔티티.  
예: “wh-question 이해 및 답변 정확도가 3개월 사이에 30% 향상” 같은 리포트 생성에 활용.

주요 속성

- task_attempt_id (PK, string)  
      
    
- activity_id (FK → Activity)  
      
    
- task_definition_id (FK → TaskDefinition, nullable): 사용 시 문항 템플릿과 연결  
      
    
- child_id (FK → Child)  
      
    
- session_id (FK → Session)  
      
    
- expected_response_pattern (enum: description, wh_question_answer, repetition, narrative, opinion, role_play 등):  
    어떤 능력을 검증하는 질문인지  
      
    
- evaluated_correctness (enum: correct, partially_correct, incorrect, off_topic, not_applicable): 정답 여부  
      
    
- related_utterance_ids (array<FK → Utterance>): 해당 답변으로 간주하는 발화들  
      
    
- instruction_understood (enum: yes, partially, no):  
    아이의 응답이 의도된 질문/지시를 이해한 흔적인지  
      
    

[Note: 언어 능력 판단 관점]

- “이 발화가 어떤 과제를 수행하는 도중 나온 말인가?”를 추적 가능  
      
    
- 이 발화가 어떤 Activity/Task에 속해 있었는지  
    (묘사, 설명, 이야기 만들기, 질문에 답하기, 의견 말하기 등)  
      
    
- 그 과제에 대해 아이가 정답/부분정답/오답/오프토픽이었는지

### 2.7 Skill (교육/발달 스킬)

설명  
서비스가 추적하는 언어/인지/정서 관련 스킬. 예: 어휘력, 문장 구성, 발음, 집중력, 감정 표현 등.

주요 속성

- skill_id (PK, string)  
      
    
- name (string): 예) Vocabulary, Pronunciation  
      
    
- category (enum: language, cognitive, emotional)  
      
    
- description (string)  
      
    
- unit (string): 예) score(0~100), level(1~5)  
      
    
- is_core_metric (bool): KPI에 직접 사용되는 핵심 스킬 여부  
      
    
- mode (enum: receptive, expressive, both): 수용언어/표현언어 구분  
      
    
- age_band (enum: 4_6, 7_9, 10_12 등): 해당 스킬 레벨의 타겟 연령대  
      
    
- can_do_examples (text/JSON): 해당 레벨에서 아이가 할 수 있는 말하기 예시  
      
    
- skill_schema_version (string): Skill 분류/정의 버전 (예: v1, v1.1 등)  
      
    

[Note: 언어 교육 관점에서 Skill 분류]

- 언어 영역 구분  
      
    

- 음운/발음(phonology/pronunciation)  
      
    
- 어휘(vocabulary)  
      
    
- 문법/문장 구성(grammar/syntax)  
      
    
- 담화/이야기 구성(discourse/narrative)  
      
    
- 화용(pragmatics: 상황에 맞게 말하기, 차례 지키기, 요청하기 등)  
      
    

- 모드 구분  
      
    

- 수용 언어(receptive): 듣고 이해하기  
      
    
- 표현 언어(expressive): 말로 표현하기  
      
    
- 필요 시 reading/writing은 후속 단계에서 확장  
      
    

- 발달 단계/레벨 정의 예:  
      
    

- Vocab_Level_2: “일상적인 주제에서 나이 또래 평균 수준의 단어를 사용함”  
      
    
- Pragmatics_Level_1: “간단한 요청은 가능하나, 차례 지키기/주제 유지가 어려움”

- child_id (FK → Child)  
      
    
- skill_id (FK → Skill)  
      
    
- snapshot_date (date)  
      
    
- raw_score (float)  
      
    
- level (enum/int): 내부 등급 체계  
      
    
- percentile_by_age (float, 0~100): 또래 대비 위치 (옵션)  
      
    
- trend_direction (enum: improving, stable, declining)  
      
    

평가 메타데이터

- source (enum: ai_auto, expert_manual, hybrid):  
      
    

- AI 자동 산출인지, 전문가 수기 평가인지, 혼합인지  
      
    

- evaluation_method (string/enum): 예) “model_xyz_v2”, “rubric_A_v1”  
      
    
- evaluation_model_version (string, nullable):  
      
    

- ML 기반 평가일 경우 모델 버전  
      
    

- skill_schema_version (string, optional):  
      
    

- 해당 SkillLevel이 어떤 Skill 정의 버전에 기반했는지 (Skill.skill_schema_version과 align)

## 3. 관계 정의

형식:

- 관계명 (R)  
      
    
- 도메인(From) → 레인지(To)  
      
    
- 카디널리티  
      
    
- 설명 / 예시  
      
    

### 3.1 주요 관계 표

|   |   |   |   |   |   |
|---|---|---|---|---|---|
|관계 ID|From (엔티티)|To (엔티티)|카디널리티|관계명 (R)|설명|
|R1|FamilyAccount|Child|1:N|has_child|한 가족 계정에 여러 아동 연결|
|R2|FamilyAccount|Subscription|1:N|has_subscription|구독 히스토리|
|R3|Subscription|SubscriptionPlan|N:1|uses_plan|구독 인스턴스가 어떤 플랜인지|
|R4|Child|Session|1:N|participated_in|아동이 참여한 세션들|
|R5|Session|Utterance|1:N|contains_utterance|세션 내 발화들|
|R6|Session|CurriculumUnit|N:1|follows_curriculum|세션이 어떤 커리큘럼 유닛 기반인지|
|R7|CurriculumUnit|Activity|1:N|contains_activity|커리큘럼을 구성하는 활동|

|R9|Utterance|Skill|N:M|measures_skill|한 발화가 어떤 스킬을 측정/반영하는지|
|R10|Child|SkillLevel|1:N|has_skill_level_snapshot|아동의 시점별 스킬 스냅샷|
|R11|SkillLevel|Skill|N:1|of_skill|어느 스킬에 대한 수준인지|
|R12|Report|Child|N:1|summarizes_child|리포트의 대상 아동|
|R13|Report|Session|N:M|summarizes_sessions|리포트가 요약하는 세션들|
|R14|Report|SkillLevel|N:M|summarizes_skill_levels|리포트에 포함된 스킬 상태|
|R15|Child|Metric|1:N (log)|has_metric_value|개인별 발달 지표 로그|
|R16|Report|ReportSkillSummary|1:N|has_skill_summary|리포트의 스킬별 요약|

## 5. 대표 질의 및 유즈케이스 (온톨로지 검증용)

엔지니어/데이터/교육 전문가가 “이 온톨로지가 쓸모 있는지”를 검증할 때 사용할 대표 질문들:

### 아동 개인 인사이트

- 지난 3개월 동안 Child X의 Vocabulary SkillLevel 트렌드는 어떻게 되는가?  
      
    
- Child X와 유사한 스킬 패턴을 가진 또래들은 어떤 Activity 조합을 많이 수행했는가?  
      
    

### 리포트 생성

- 이번 달 Child X의 모든 Session, Utterance, SkillLevel, TaskAttempt를 묶어서  
    “월간 학습 활동 요약 / 강점 / 개선 영역 / 다음 달 추천”을 생성할 수 있는가?  
      
    

### 교육 효과 측정

- 서비스 사용 3개월 이상 아동 중, 어휘력 향상 20% 이상을 달성한 비율은?  
      
    
- 첫 달 대비 발화량 50% 이상 증가한 아동의 공통 특성은?  
      
    
- 특정 Skill의 evaluation_model_version이 바뀐 시점을 경계로,  
    지표 해석을 어떻게 보정할 수 있는가?  
      
    

### 비즈니스 KPI

- Plan별 Churn Rate, LTV/CAC, MAU 추이 분석 시 필요한 관계/엔티티가 빠짐없이 연결되는가?  
      
    
- Billing Platform(Direct vs Apple/Google) 별로 Retention/Engagement 차이를 분석할 수 있는가?

## 6. 비즈니스 룰 및 제약 조건 (초안)

예시로 정의하는 규칙 (추후 OWL/Rule 엔진에 반영 가능):

### 아동-가족 제약

- 한 Child는 반드시 하나의 FamilyAccount에 속해야 한다.  
      
    

### 세션-아동 제약

- 모든 Session은 정확히 한 명의 Child와 연결되어야 한다.  
      
    

### 리포트 제약

- 월간 Report는 최소 1개 이상의 Session 및 SkillLevel을 참조해야 한다.  
      
    

### 스킬/레벨 제약

- 하나의 (child_id, skill_id, snapshot_date) 조합은 유일해야 한다.  
      
    

### 플랜/구독 제약

Subscription이 active가 아니면 Session 생성 불가 (assessment/trial은 예외 규칙 별도 정의).  
  
[정의] “활성 상태(active)”란  
  
status IN ('paid_good_standing', 'paid_pending_cancel', 'grace', 'trial')

 을 의미한다.  
즉, 'stopped', 'hold' 상태는 inactive로 간주한다.

## 7. 전문가 검증 포인트

향후 아래 역할들이 본 온톨로지를 검토하고 수정 제안:

### 언어치료사 / 교육 전문가

- 언어·발달 Skill 체계의 교육학적 구체화  
      
    
- Utterance/Session/TaskAttempt 지표가 실제 임상/교육 평가 프레임워크와 얼마나 align되는지  
      
    
- “수용 언어(듣기)”와 “표현 언어(말하기)” 분리가 적절한지  
      
    
- Report / ReportSkillSummary에 교육적으로 의미 있는 목표·예시·지도 전략이 충분히 담길 수 있는지  
      
    
- 음성 데이터 보존 전략이 평가 품질 vs 프라이버시/비용 관점에서 적절한 균형인지  
      
    

### 데이터 사이언티스트 / 분석가

- 교육 효과 지표(어휘력 향상, 발화량 증가 등)를 계산하는 데 필요한 데이터가 충분히 모델링되어 있는지  
      
    
- STT/스코어링 모델 버전 변경 시, 지표의 시계열 해석에 문제가 없도록 메타데이터가 충분한지  
      
    
- Graph 기반 추천/분석/진단 모델 설계에 필요한 관계/피처가 누락되지 않았는지  
      
    

### 플랫폼/백엔드 엔지니어

- 실제 저장소(PostgreSQL + GraphDB + Object Storage(S3 등)) 설계에 무리가 없는지  
      
    
- 성능/확장성 측면에서 쿼리 복잡도, N:M 관계 처리, 이벤트 스트림 처리 구조가 적합한지  
      
    
- PII/민감 데이터에 대한 접근 제어/마스킹/로그 정책을 어떻게 적용할지

## 9. 부록: 리포트 Metric – Skill 매핑 정의

이 부록은 **월간 리포트(Report)** 에서 사용하는 핵심 지표(Metric)를
도메인 온톨로지 상의 `Skill`, `SkillLevel`, `Session`, `Utterance`, `TaskAttempt` 와 어떻게 연결할지에 대한 설계를 정의한다.

목적:

* 부모에게는 4~5개의 직관적인 섹션으로 요약된 리포트 제공
* 내부적으로는 `Skill`, `SkillLevel`, `Metric` 간의 명시적인 매핑을 통해

  * 교육 효과 측정
  * 추천/진단 모델 피처 생성
  * 장기적인 성장 추적
    을 일관된 구조로 지원

### 9.1 리포트 상위 섹션 구조

부모에게 노출되는 월간 리포트의 상위 섹션은 다음과 같이 정의한다.

1. **말하기 자신감 & 참여도 (Speaking Confidence & Participation)**
2. **어휘력 & 문장력 (Vocabulary & Sentence Building)**
3. **이야기 구성력 (Storytelling Skills)**
4. **듣고 이해하기 (Listening & Understanding)**
5. **학습 습관 & 루틴 (Learning Habits)**

각 섹션은 1개 이상의 `Metric` 엔티티와 연결되며,
각 Metric 은 하나 이상의 `Skill` / 행동 신호(Session, Utterance, TaskAttempt 기반 집계값)를 입력으로 사용한다.

### 9.3 Metric 정의 (MVP)

`Metric` 엔티티는 리포트 및 서비스/교육 효과 측정에 사용되는 정량 지표를 정의한다.

```text
Metric
- metric_id (PK, string, snake_case)
- name (string)
- category (enum: service, business, education)
- description (text)
- calculation_formula (text)   // 논리/의사코드 수준으로 정의
- unit (string)                // 예: score_0_100, level_1_5, category, ratio 등
- target_value (float, nullable)
```

아래는 **교육(education)** 카테고리에 해당하는,
리포트 핵심 Metric 정의이다.

#### 9.3.1 말하기 자신감 & 참여도 영역

| 필드          | 값                                       |
| ----------- | --------------------------------------- |
| metric_id   | `speaking_confidence_score`             |
| name        | Speaking Confidence                     |
| category    | `education`                             |
| unit        | `score_0_100`                           |
| description | 아이의 말하기 자신감 및 자발적인 발화 수준을 종합적으로 나타내는 점수 |

**calculation_formula (개념 정의)**

* 입력:

  * `SkillLevel(self_expression_willing).raw_score`
  * 월간 자발 발화 비율:
    `spontaneous_utterance_count / total_child_utterance_count`
  * 평균 발화 길이:
    `avg(word_count)` for child utterances
  * “pass/패스” 응답 비율

* 개념적 계산 예시:

  * 위 입력들을 0~1 범위로 정규화 후,
    `speaking_confidence_score = 0.5 * self_expression_score + 0.3 * spontaneous_ratio + 0.2 * normalized_avg_word_count - penalty_for_high_pass_ratio`
    형태로 0~100 점수로 변환

---

| 필드          | 값                                    |
| ----------- | ------------------------------------ |
| metric_id   | `participation_consistency`          |
| name        | Participation & Consistency          |
| category    | `education`                          |
| unit        | `score_0_100`                        |
| description | 세션 참석률, 세션 내 참여도, 집중도를 종합적으로 나타내는 지표 |

**calculation_formula (개념 정의)**

* 입력:

  * 월간 세션 수: `session_count_in_period`
  * 목표 세션 수: `target_session_per_week * weeks_in_period`
  * 평균 `Session.engagement_score`
  * 평균 `Session.attention_score`
* 개념적 계산:

  * 출석률 = `min(1, session_count / target_session_count)`
  * 참여 스코어 = `avg(engagement_score, attention_score)`
  * 최종 스코어 = 가중 합 후 0~100으로 스케일링

#### 9.3.2 어휘력 & 문장력 영역

| 필드          | 값                                    |
| ----------- | ------------------------------------ |
| metric_id   | `vocabulary_level`                   |
| name        | Vocabulary Level                     |
| category    | `education`                          |
| unit        | `level_1_5` (또는 score_0_100)         |
| description | 일상 대화에 사용되는 영어 어휘의 폭과 사용 빈도를 나타내는 지표 |

**calculation_formula (개념 정의)**

* 입력:

  * `SkillLevel(vocab_core).raw_score`

  * type-token ratio 등 보조 지표
* 개념적 계산:

  * `base = normalized(SkillLevel(vocab_core).raw_score)`
  * `usage = normalized(unique_word_count_per_minute)`
  * `vocabulary_level = f(base, usage)` 를 1~5 또는 0~100으로 변환

### 9.4 Metric ↔ SkillLevel 연산 규칙

1. **SkillLevel 갱신 주기**

   * 최소 월 1회, 리포트 생성 직전에 `SkillLevel` 스냅샷 생성
   * `snapshot_date` 는 리포트 기간 종료일 또는 직후 날짜로 설정

2. **Metric 계산 시 기본 원칙**

   * 언어 역량 중심 Metric:

     * 대응 `SkillLevel.raw_score` 를 기본 스코어로 사용
     * Utterance / TaskAttempt 기반 관측치를 보조 신호로 사용
   * 행동/참여 중심 Metric:

     * `Session`, `Utterance`, `TaskAttempt` 집계값을 주된 입력으로 사용
     * 필요 시 관련 SkillLevel 과 조합

3. **정규화 및 버킷화**

   * 내부 계산은 가급적 0~1 / 0~100 스코어로 정규화
   * 부모 리포트에는 level(1~5) 또는 3단계 버킷(high/mid/low, good/fair/weak) 형태로 노출

### 9.5 ReportSkillSummary 생성 규칙

`ReportSkillSummary` 엔티티는
각 `Skill` 에 대한 부모 친화적 요약 문장을 저장하기 위한 구조이다.

```text
ReportSkillSummary
- report_skill_summary_id (PK, string)
- report_id (FK → Report)
- skill_id (FK → Skill)
- level (enum/int)               // 해당 시점 SkillLevel.level 복사
- summary_for_parent (text)      // 부모용 자연어 설명
```

#### 9.5.1 버킷 기반 텍스트 템플릿

각 Skill 마다 level 또는 percentile에 따른 버킷(high/mid/low)을 정의하고,
버킷별 텍스트 템플릿을 미리 준비해둔다.

예: `vocab_core` (어휘력)

* high (예: level 4~5 또는 percentile ≥ 70)

  * “[이름]이는 또래에 비해 다양한 영어 단어를 사용하고 있어요.
    최근 4주 동안 새로운 단어를 자연스럽게 말 속에 섞어 쓰는 모습이 자주 보입니다.”
* mid

  * “[이름]이는 일상 대화에 필요한 기본 단어들을 안정적으로 사용하고 있어요.
    앞으로는 좋아하는 주제(예: 동물, 음식)를 중심으로 단어 폭을 조금씩 넓혀가는 연습을 함께 해볼 수 있어요.”
* low

  * “[이름]이는 아직 익숙한 단어 위주로 말하는 편이에요.
    루나와의 대화에서 새 단어를 들었을 때, 포포의 도움으로 한 번 더 따라 말해보면 어휘력이 더 빨리 자랄 수 있어요.”

다른 Skill(`sentence_basic`, `narrative_simple`, `wh_question_response`, `self_expression_willing` 등) 도
동일한 방식으로 high/mid/low 템플릿을 정의하여
리포트 생성 시 자동 매핑한다.

### 9.6 추후 확장 포인트

* Metric 확장

  * 정서/행동 중심 Metric (감정 표현, 자기조절 등) 추가
  * B2B용 학급/학교 수준 집계 Metric 추가
* Skill 계층 구조

  * `Skill` 간 상위/하위 관계(예: language → vocabulary → animal_words) 도입 가능
* 다언어 확장

  * `primary_language != ko` 인 케이스에 대한 별도 벤치마크 스케일 도입
