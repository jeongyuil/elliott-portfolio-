
# 커리큘럼 스토리텔링 프롬프트 라이브러리 (Curriculum Storytelling Prompt Library)

## 목적 (Purpose)
이 문서는 커리큘럼 스토리, 캐릭터 행동, 학습 흐름 및 진행 로직을 설계하고 발전시키는 데 사용되는 프롬프트들을 수집하고 체계화한 것입니다.
FDD(기능 명세서)의 일부로서 재사용, 확장 및 버전 관리가 가능하도록 의도되었습니다.

---

## 1. 상위 레벨 커리큘럼 설계 프롬프트 (High-Level Curriculum Design Prompts)

### 1.1 커리큘럼 아크(Arc) 정의
**Prompt**
```
4주차 커리큘럼을 설계하시오. 각 주차는 서사적 연속성, 점진적인 스킬 숙련, 그리고 정서적 몰입을 구축해야 합니다.
각 주차별 학습 목표, 정서적 톤(Tone), 그리고 학습자의 변화(Transformation)를 정의하시오.
```

### 1.2 학습 성과 백캐스팅 (Backcasting)
**Prompt**
```
4주차 말에 도달해야 할 최종 학습자 역량을 기준으로,
주차별 마일스톤과 선행 스킬을 역순으로 설계(Backward-design)하시오.
각 주차가 측정 가능한 행동적 성과를 포함하도록 하시오.
```

---

## 2. 스토리 & 세계관 구축 프롬프트 (Story & Worldbuilding Prompts)

### 2.1 세계관 설정 (World Setting)
**Prompt**
```
상호작용을 통해 스킬을 습득할 수 있는, 허구적이지만 설득력 있는(Grounded) 학습 세계관을 창조하시오.
현실 세계의 학습 원리를 반영하는 세계의 규칙, 진행 로직, 그리고 제약 조건을 정의하시오.
```

### 2.2 서사 일관성 체크 (Narrative Consistency Check)
**Prompt**
```
커리큘럼 스토리라인을 검토하고, 주차 간의 서사 논리, 동기 부여, 또는 캐릭터 행동에서의 단절을 식별하시오.
학습 의도를 보존하면서 이를 해결할 수 있는 수정안을 제안하시오.
```

## 3. 캐릭터 디자인 & 행동 프롬프트 (Character Design & Behavior Prompts)

### 3.1 친구 캐릭터 — Luna (살아있는 외계 아이)
**Prompt**
```
우주에서 온 살아있는 아이(Luna)를 디자인하시오. 로봇이 아닌 살아있는 생명체로,
주인공(아이)과 비슷한 나이대이다. 호기심이 많고 장난기가 있으며,
지구 문화에 대해 귀여운 실수를 한다.
아이(주인공)가 Luna에게 영어를 가르쳐주는 역멘토링(Reverse Mentoring) 구조를 형성한다.
성격 특성, 진정한 감정 표현(놀라움, 감사), 그리고 아이의 도움으로
Luna의 이해가 어떻게 성장하는지 정의하시오.
```

### 3.2 코치 캐릭터 — Popo (비밀 요원 코치)
**Prompt**
```
정부에서 파견된 친절한 비밀 요원(Popo)을 디자인하시오.
Popo는 아이를 사랑하는 따뜻한 마음의 소유자이며, 훌륭한 코치 역할을 한다.
Popo는 아이를 **"캡틴"**이라고 부르며, 주인공으로서의 자부심을 강화한다.
정서적 반응, 코칭 스타일, 개입 타이밍 vs. 관망 타이밍,
그리고 한국어와 영어를 아이에게 어떻게 연결해주는지 정의하시오.
```

### 3.3 트라이어드 상호작용 (Luna + Popo + 아이 = 주인공)
**Prompt**
```
Luna(친구), Popo(코치), 아이(주인공/캡틴)가 함께 상호작용하는 대화를 작성하시오.
역할 분담을 명확히 하시오:
- Luna = 호기심 & 실수 (영어 only)
- Popo = 코칭 & 정서적 안전 (한국어 + 영어)
- 아이 = 주인공/캡틴, Luna를 가르치는 선생님
아이는 수동적 학습자가 아니라 이야기의 **주인공**으로 느껴야 한다.
```

## 4. 세션 레벨 스토리 프롬프트 (Session-Level Story Prompts)

### 4.1 세션 오프닝 — 동화책 스타일 (Session Opening)
**Prompt**
```
다음 3단계 Interactive Story Flow를 포함하는 세션 오프닝을 작성하시오:

1. [Phase 1: Narrator — 동화책 나레이션]
   - "옛날 옛적에~" 식으로 시작하는 동화책 읽어주기 형식 (TTS용 스크립트)
   - 세계관 설정, 이전 줄거리 요약, 현재 상황 설명
   - 목적: 몰입감을 조성하여 아이가 이야기 세계로 자연스럽게 진입
2. [Phase 2: Transition — 주인공 진입]
   - Popo가 아이를 **"캡틴"**이라 부르며 이야기 속으로 초대
   - 참여를 시작하기 위한 트리거 단어 (예: "Go!", "Open!", "Start!")
3. [Phase 3: Interaction Start — 주인공의 모험 시작]
   - 아이(주인공/캡틴)가 Luna와 첫 만남 — Luna가 진심으로 반가워함
```

### 4.2 성찰 & 마무리 (Reflection & Closure)
**Prompt**
```
학습을 강화하고, 노력을 인정하며, 다음 세션의 도전을 암시하는 성찰적 마무리 장면을 창조하시오.
```

## 5. 스킬 통합 프롬프트 (Skill Integration Prompts)

### 5.1 스킬-스토리 매핑 (Skill-to-Story Mapping)
**Prompt**
```
주어진 스킬 목록을 바탕으로, 노골적인 설명 없이 스토리 이벤트, 대화, 또는 학습자의 선택지 속에 각 스킬을 자연스럽게 녹여내시오.
```

### 5.2 난이도 조정 (Difficulty Calibration)
**Prompt**
```
이전 스킬 습득 수준에 비추어 볼 때 이번 세션의 과업이 너무 쉽거나 어려운지 평가하시오.
서사(Narrative)에 기반한 조정 방안을 제안하시오.
```

---

## 6. QA & 일관성 프롬프트 (QA & Consistency Prompts)

### 6.1 인지 부하 검토 (Cognitive Load Review)
**Prompt**
```
이 세션을 검토하여 인지 과부하(Cognitive overload) 요소를 확인하시오.
지시사항, 스토리, 또는 과업을 단순화하거나 분할해야 할 부분을 식별하시오.
```

### 6.2 FDD 정합성 확인 (Alignment with FDD)
**Prompt**
```
이 커리큘럼 콘텐츠를 FDD(기능 명세서)와 대조하여 확인하시오.
누락된 요구사항, 잘못 정렬된 스킬, 또는 범위 확장(Scope creep)을 표시하시오.
```
